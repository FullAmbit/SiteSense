Site Management

1. Dashboard

The Dashboard is the first thing an administrative user sees upon logging into the Control Panel. Here information is displayed about the server and current SiteSense installation.

Information Displayed

  • SiteSense Version
  • Server Time
  • Server Signature
  • Server Name
  • Server Address
  • Gateway Interface
  • Server Protocol
  • PHP Version
  • Zend Version
  • Host OS

2. Settings

SiteSense is highly customizable and with that comes many different options. To make things easier on new users, SiteSense installs with all settings in a default configuration. This allows you to quickly power up SiteSense and get the hang of things.

2.1 Global Settings

  • Site Title
    Constructs the TITLE tag and in most skins the content of the top level heading (H1)
  • Default Theme
    Determines which of your installed themes users will default to
  • Default Time Zone
    Sets the default time zone for all newly registered users
  • Homepage
    Sets the landing "page" for visitors of your site
  • Hide Content From Guests
    Sets the CMS to show either the login page or the registration page instead of the normal content to guests
  • Use BB Code
    Enables Bulletin Board Code to format blog posts and comments
  • WYSIWYG Editor
    Select the editor you'd like to use to edit dynamic HTML content. By default, ckeditor is used.
  • Default Blog
    Sets the default blog
  • Blog Items Per Page
    Determines how many posts are shown on a list page and the number of news items shown on the homepage
  • Footer Content
    Content to be displayed in the footer of each page on the site

2.2 Advanced Settings

  • Character Encoding
    Sets the Content-Encoding meta-tag and is used by PHP's "header" function to set the mime-type in the http response header
  • Use compression?
    Turns on and off server gzip/mod_deflate compression if available
  • Compression Level
    If compression is on and enabled, sets the amount of compression used
    • 1 - least compression / low CPU load
    • 9 - most compression / high CPU use
  • Default Session Timeout (seconds)
    Sets how long a user account is allowed to remain logged in without requesting any pages
  • Use modRewrite (.htacess)
    This will enable support for what is commonly referred to as "SEO friendly URLs." If enabled this will build all links to assume that the rewrite engine is in use and redirecting all calls to the system to index.php directly. In order to use this feature make sure that the SiteSense root directory contains an .htaccess file including something like this:
            RewriteEngine On
        RewriteRule !\.(gif|jpg|png|css|js|swf|html|ico|zip|rar|pdf|xml|mp4|mpg|flv|mkv)$ index.php
    You will want to make sure any and all data types you do not want the CMS to handle have their file extensions listed above.

2.3 Registration Settings

  • Default User Group
    Determines which user group newly registered users will be associated to.
  • Verify EMail
    New users will be required to click a link that was emailed to their address specified during registration in order to complete registration.
  • Require User Activation?
    New users must be manually activated by an administrator.

2.4 Content Delivery Network Settings

  • Use CDN?
    Enable or disable the use of a CDN (content delivery network).
  • CDN Plugin
    Sets the plugin used for accessing the CDN
  • CDN Base Directory
    Sets the base directory to upload to on your CDN server (Use trailing and base slashes)
  • Large CDN URL
    Sets the URL of your larger CDN server where files are pulled from the disk
  • Small CDN URL
    Sets the URL of your smaller CDN server where files are pulled from the RAM
  • Flash CDN URL
    Sets the URL of your flash streaming CDN server where files are pulled from the disk.

3. Main Menu

Navigation is built into SiteSense's core through the Main Menu Module. Here main menu items can be added, modified, and removed. The Main Menu Module is deeply integrated into the core modules meaning that main menu items can be generated automatically for you upon the creation of other entities, such as pages and forms.

Properties

  • Text
    Sets the text that a visitor will see on the menu item
  • Title
    Sets the tooltip text that a visitor will see when they hover over the menu item
  • URL
    Sets the URL link that the menu item points to, based on the root of your SiteSense install.
  • Side
    Determines which side of the menu your menu item will display on if your theme supports this functionality.
  • Enable
    Determines if the menu item will be displayed. Disabled items do not appear to users.
  • Parent Menu Item
    Menu items can be associated via a parent/child relationship. Set to "--none--" to make it a top-level menu item.

Controls

  • Enable
    Toggles the enable property of a main menu item
  • Delete
    Removes the main menu item
  • Switch Side
    Toggles the side of the main menu item (left/right)
  • Move Up (⇑) / Move Down (⇓)
    Changes the ordering of main menu items

4. Pages

The Pages Module allows for the management of pages that are primarily used for the display of static content. While their content may be static, they can easily and dynamically be modified using the Control Panel.

Properties

  • Name
    The page name is the URL entity of a page and therefore is required to be unique.
  • Page Title
    The page title populates the heading tag for the page.
  • Parent Page
    Pages can be associated via a parent/child relationship. If sidebar navigation is enabled this sub-page will appear in a separate menu off the parent.
  • Create Main Menu Link
    When checked, it will generate a main menu item for the page. This option only appears when creating a new page.
  • Parent Menu Item
    This option lets you choose a parent menu item for the menu item for the page, if you've enabled this option above.
  • Menu Link Text
    The menu link text is what will be used as the Text property for the automatically created main menu item (if enabled).
  • Page Content
    Page content is the content that will go inside the automatically generated 'contentBox'. HTML is allowed in here, though any sub-headings you wish to use should start at H3, since 'Page Title' is used as the h2.
  • Live
    Determines if the page will be displayed. Check the box if you want the page to be live on the web.

Controls

  • Move Up (⇑) / Move Down (⇓)
    Changes the ordering of pages within their parent page
  • Sidebars
    Manage the sidebars for the selected page. If no sidebars are chosen specifically, the sidebars selected for the "pages" module as a whole will be used.
  • Add Child
    Brings up the form to add a child form with the selected form as its parent.
  • Delete
    Removes the page

5. Blogs

SiteSense's core includes a feature-filled blogging solution ready for seamless integration into your site. With the Blogs Module you can host multiple different blogs on a single SiteSense installation.

Blogs are made up of posts which can be organized into categories. Users add comments to posts, and administrative users can approve or disapprove comments for display.

The Blogs Module also generates an RSS feed for every blog entity. http://example.com/blogs/{BlogShortName}/rss

Properties

  • Blog Name
    The name used for creating the URL. It must be unique.
  • Blog Title
    The blog title populates the heading tag for the blog.
  • Blog Owner
    The owner of a blog will have access to their own blog even if they don't have Blogs->Access Others permission. They are still required to have the permission pertaining to any action they wish to perform (post/edit/reply/approval).
  • RSS Channel Image
    Image to be displayed with the RSS feed
  • Managing Editor
    Sets the managing editor email that will be displayed on the RSS feed
  • Web Master
    Sets the webmaster email that will be displayed on the RSS feed
  • Comments Require Login
    Forces readers to log-in before commenting on a blog
  • Top Level Blog
    Automatically generates a URL Route to allow access to the form by /{BlogName} as an alternative to /blogs/{ BlogName}
  • Allow Comments
    Determines if commenting is enabled on the blog
  • Number Per Page
    Sets the number of post entries to show on a page
  • RSS Link Override
    Entering a link here will change the links of the "Subscribe" buttons on the blog pages. This is useful, for example, for a Feedburner URL.
  • Description
    Short text to describe this blog (Not currently displayed to visitors)

Controls

  • New Post
    Adds a new post to the selected blog
  • Modify Blog
    Edit the properties of the blog
  • Delete
    Removes the blog and all of its contents (posts/comments)

5.1 Blog Posts

Properties
  • Post Name
    The name used for creating the URL. It must be unique.
  • Post Title
    The post title populates the heading tag for the post.
  • Tags
    Assigns tags to the post (Tags must be comma separated)
  • Categories
    Posts can be placed into a single category or no category at all for organizational purposes.
  • Summary
    A brief description of the content of the post
  • Content
    The content of the post you are writing/editing can be created from within the WYSIWYG editor
  • Allow Comments
    In order to allow comments on a post, the "Allow Comments" checkbox must be checked for both the parent blog and the post.
  • Live
    Determines if the page will be displayed
Controls
  • Edit Comments
    Lists all comments awaiting approval, approved comments, and disapproved comments. Allows you to change the status of comments and to remove comments
  • Delete
    Removes the post and all related content (comments)

5.2 Blog Post Comments

Properties
  • Edit Comments
    Lists all comments awaiting approval, approved comments, and disapproved comments. Allows you to change the status of comments and to remove comments
  • Delete
    Removes the post and all related content (comments)
Controls
  • Approve / Disapprove
    Toggles the status of the comment (Only approved comments are visible to visitors)
  • Edit
    Edits the properties of a comment
  • Delete
    Removes a comment

5.3 Blog Post Categories

Properties
  • Name
    The name is the category's title that will be displayed to visitors
Controls
  • Modify Category
    Edits the properties of a category
  • Delete Category
    Removes a category

6. Dynamic Forms

Forms can be dynamically created from within SiteSense's Control Panel without the need to write a single line of code. From the Control Panel, forms and fields can be added, modified, and removed and submit data can be outputted into a table.

Properties

  • Name
    The name used for creating the URL. It must be unique.
  • Title
    The form title populates the heading tag for the form.
  • Submit Title
    Sets the text displayed on the submit button
  • E-Mail
    Specify an email address to send submitted form data to. Multiple addresses should be separated by commas with no spaces
  • API Hook
    If you'd like to send data from this form to an API (such as for a newsletter signup or survey service), select the plugin which you would like to use to contact the API here
  • Create Menu Link
    When checked, it will generate a main menu item for the page.
  • Menu Title
    The menu link title is what will be used as the visible text for the automatically created main menu item (if enabled).
  • Raw Content Before
    The content to be displayed before the form can be entered from within the WYSIWYG editor.
  • Raw Content After
    The content to be displayed after the form can be entered from within the WYSIWYG editor.
  • Success Message
    This is the message that users will see after successfully submitting the form.
  • Require Login?
    Determines if users must be logged in prior to submitting the form
  • Top Level
    Automatically generates a URL Route to allow access to the form by /{FormName} as an alternative to /dynamic-forms/{ FormName}
  • Enabled
    Determines if the form will be displayed to visitors

Controls

  • Edit Settings
    Edit properties of the form
  • Manage Fields
    Allows you to create, modify, and remove fields from the form (See Fields)
  • View Data
    Displays values from form submissions in a table
  • Sidebars
    Manage the sidebars for the selected form
  • Delete
    Removes the form and all related content (fields and data)

6.1 Fields

Properties
  • Name
    The name of the field will populate the label text and is visible to visitors
  • Description
    Explanation of the data you are trying to collect. If your theme supports it, this can be displayed to visitors upon focus of the form field.
  • Type
    The type of form element in which you are trying to create.
    • textbox - A single-line text input field.
    • textarea - Room for multiple lines of text input.
    • checkbox - A simple checkbox - useful for yes/no inputs.
    • select - A drop-down item, from which one option can be selected. You can add options to the select field after you save it.
    • timezone - A "select" form element which is pre-populated with a list of all timezones.
    • password - Input into this field type will be obsfucated and will not be saved to the database like other fields are.
  • API Field
    If you have an API Hook enabled for this form, setting the API field will determine the name of the variables sent to the API. For example, if you have a standard newsletter plugin like the CampaignMonitor or MailChimp plugins provided by SiteSense, the API Field for the e-mail address is "email" and the API Field for the name attribute is "name".
  • Required
    Determines if the field is required to be filled out prior to submission
  • Enabled
    Determines if the field should be displayed
  • E-Mail Validation?
    When checked, the value submitted will be validated as an email address. If the value is not in the proper format, it will display an error to the user. This will only work with textboxes.
  • Compare To
    The value of the field will be compared to the value of the field selected here. If they do not match, an error will be displayed to the user. This is useful when trying to confirm an email or password submission.
  • Module Hook
    If a module is selected, that module will be responsible for processing the submitted field data and it will not be viewable through Dynamic Forms.
Controls
  • Edit
    Modifies the properties of a field
  • Delete
    Removes the field from the form
  • Move Up (⇑) / Move Down (⇓)
    Changes the ordering of fields within their parent form

7. Sidebars

Sidebars contain static or dynamic content that can be displayed on the edges of almost any page within SiteSense. Dynamic content is achieved through the use of blocks which are PHP files containing code within a module. Dynamic forms and pages can have their own entity-specific sidebars, which are modified in the module-specific control panels. To get sidebars to display on the user-end, you must enable them for each module you want them to appear with through the Modules control panel in the admin zone.

Properties

  • Name
    Used only for reference in the admin control panel. Must be unique.
  • Title
    The title is the text that will be displayed above the sidebar and is visible to visitors
  • Title URL
    Optionally allows the title to be a hyperlink. Should be a full URL (If the first character is a vertical break (|) then the 'linkRoot' will be appended before it — in other words a local link in the CMS.)
  • Side
    Determines the side of the theme a sidebar will display on
  • Context Text
    The content of the sidebar you are writing/editing can be created from within the WYSIWYG editor. Dynamic sidebars can be created through the integration of blocks or other dynamic values. A block can be called by using the following syntax:
              |block:{ModuleName}_{BlockName}({Attributes})|
          Ex. |block:blogs_lastestPosts(5,1)|
    This will be replaced with a list of the five latest posts from the blog with an ID=1.

Controls

  • Move Up (⇑) / Move Down (⇓)
    Changes the ordering of sidebars
  • Delete
    Removes the sidebar
  • Switch Side
    Toggles the side of the page which the sidebar will display on, if your module supports it.

8. Modules

Modules are where most of the logic for end-user functionality resides and each module contains a particular set of functionality as indicated by the name of the module. It is important that a module be enabled before attempting to utilize its functionality in the Control Panel or on the front-end of the site.

Controls

  • Enabled / Disable
    Toggles the status of the selected module
  • Modify
    Manage the sidebars for the selected form
  • Update Translations
    Updates translations for the modules in all languages except for English.
  • Install Phrases
    Adds new phrases from the English language pack

9. Plugins

While a module largely adds functionality by utilizing the SiteSense framework, a plugin connects third party functionality by creating a bridge or interface with SiteSense. Plugins must be enabled for each module that will utilize the plugin.

Controls

  • Enabled / Disable
    Toggles the status of the selected plugin
  • Modify
    Select the modules that have access to the plugin

10. URL Routing

All page requests are routed through the URL module. Patterns will be matched from top to bottom, stopping at the first match. Once a match has been found, the pattern will be replaced with the specified replacement. Replacement URLs will be force-redirected to their matches as well, to prevent duplicate content for different URLs.

Properties

  • Pattern
    Everything after the SiteSense path in the URL will be matched to this field.
  • Replacement
    Sets what the matched pattern will be replaced with
  • Hostname
    Sets a specific hostname (domain) for the URL remap. Hostnames can be added through the hostnames module. Setting it to global will match the pattern across any of your domains.
  • Redirect
    Forces a Header 301 permanent redirect to the replacement URL.
  • Use Regular Expressions
    When checked, the pattern and replacement will utilize regular expressions. (Advanced Users Only)

Controls

  • Move Up (⇑) / Move Down (⇓)
    Changes the ordering of URL Remaps. Ordering is important here since the framework will process the list of URL remaps from top to bottom, stopping at the first match. Therefore, your most specific matches should go first and least specific matches last.
  • Modify
    Edit the properties of the URL Remap
  • Delete
    Remove the URL Remap

11. Languages

Languages allow support for your international users. Phrases allow you to output static content in any language you have installed. They can be specific to a module or global throughout the site. Each phrase must exist for each language. To enforce this, all phrases that do not first exist in English, cannot be modified or added. Thus, English is your base language. Modules that have language support enabled will also have separate tables to contain language-specific input.

Properties

  • Default
    The language that viewers first see on your website.

Controls

  • Make Default
    Sets the language as the default for viewers on your site.
  • Install Phrases
    This option is only available to the English language. Depending on the action you take, it will either add/update new/existing phrases for the English language. The script will also check to see if each phrases exists throughout the rest of your languages. If it is not found, it will add the phrase and default it to the English value.
  • Update Translations
    This option runs for a specific installed language, except for English. It will go through the phrases available to the language file-system, and add/update them accordingly. However, it will only work on phrases that have an English counterpart. In addition, phrases will only be UPDATED if they are set to their English default value. This prevents the script from removing any changes you might've manually made.

11.1 Update Translations

Properties
  • Update Modules
    Setting this will also process phrases specific to your modules.

11.2 Modify Phrases

Controls
  • Add A Phrase
    Add a new phrase. It will automatically be added to each language, however you need to provide at least an English value for the phrase. The phrase must be unique to the module and whether it belongs on the administrator panel, or the user end.
  • Modify
    Edit an existing phrase. Again, you will need to provide at least an English value for the phrase. When you modify a phrase, your edit be marked as an override. This means that future updates to that module will not affect that particular phrase, so your edits will stay in place until you decide to change them yourself. You can quickly view modified phrases by clicking "Modify Overrides" at the top of the languages page in the admin control panel.
Properties
  • Phrase
    The name of the phrase. This is also how you would refer to it in the code.
  • Text
    The value of the phrase for the language you are working in.
  • Module
    The module to which the phrase belongs. Phrases can also be global and will be loaded on every page of the website.
  • Level
    Phrases can belong on either the administrator panel or the user end. This is also a unique parameter for the phrase.

11.3 Modify Overrides

Controls and properties on the Modify Overrides are the same as those on the Modify Phrases page, documented above.
Modify Overrides allows you to quickly view and edit phrases which you have modified in the past.

12. Hostnames

Hostnames allow a single installation of SiteSense to have a unique group of settings across different domains. You do not need to add a hostname if you are only using your website on one domain.

Properties

  • Theme
    The theme that people will see when accessing your website from this hostname.
  • Language
    The default language that will be displayed to people browsing this hostname.
  • Homepage
    The first page that users will see when coming to this hostname.

Controls

  • Add A Hostname
    Add a specific domain to use this installation of SiteSense.

User Management

1. Users

Properties

  • Personal Information
    The following information can be associated to a user via the Users Module. (* marks a required field)
    • * First Name
    • * Last Name
    • * Username
    • Contact E-Mail
    • Public E-Mail
    • * Password
    • * Default Time Zone (Defaults to Global Time Zone in Settings)
  • User Groups
    Users are placed into groups, and groups contain permissions. A user can be assigned to multiple groups or no group at all. Furthermore, a user can be temporarily added to a group for a limited time by setting an expiration time.
  • User Specific Permissions
    Each permission defaults to "Inherited", meaning that the user will only be granted the permission if a group the user is associated with has the permission. You can also set a permission as "Allowed" or "Disallowed" which will grant or deny, respectively, the user the permission regardless of whether or not they have the permission already.

2. Permissions

Users can be added to multiple groups or to no group at all, and groups can be assigned permissions. Users can also be individually set to allow a permission, inherit the permission if already granted by an associated group (neutral), or have the permission marked as revoked regardless of whether or not the user has already been granted the permission from an associated group. A user's list of granted permissions is the sum of all the permissions from associated groups, plus all permissions granted on an individual basis, and minus all permissions revoked from the specific user.

It can be difficult to tell outright which permissions a user does and does not have. To check the overall verdict of the permissions of a user, simply go to your administrator control panel and edit the user. Click on any form field in the Permissions section and a box will appear to the right, saying "Verdict: Allow/Forbid/Neutral". That box will display the end result of the permissions system logic.
